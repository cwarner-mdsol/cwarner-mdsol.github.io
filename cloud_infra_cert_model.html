Cloud Infrastructure certificate model

Certificate issuance

Our model is pretty straight-forward we install certbot.

"Certbot is an easy-to-use automatic client that fetches and deploys SSL/TLS certificates for your webserver. Certbot was developed by EFF and others as a client for Let’s Encrypt and was previously known as “the official Let’s Encrypt client” or “the Let’s Encrypt Python client.” Certbot will also work with any other CAs that support the ACME protocol."

In this case we follow the instructions for our distro, where we grab certbot from a launchpad ppa (which is run by Canonical):

$ sudo apt-get install software-properties-common

$ sudo add-apt-repository ppa:certbot/certbot

$ sudo apt-get update

$ sudo apt-get install certbot

Certbot does the initial request for the certificate based on data that we provide and then is setup to automate renewal. This can be automated such that it will request and renew a certificate for a certain property (whatever_here.imediata.net). In this case we've done so for vault.imedidata.net and have the cert/keys placed in /etc/letsencrypt/live/vault.imedidata.net/fullchain.pem and /etc/letsencrypt/live/vault.imedidata.net/privkey.pem respectively.

Our vault application is then configured to use this cert/key combo for TLS and that's all there is to it. We don't have to modify any root stores because the Let's Encrypt CA is signed by IdenTrust which is already available in most major operating systems ssl root stores.

Cert Revocation

We can revoke a cert at anytime, for any reason and have one summarily issued again. TLS Cipher Suite compromised? No problem. Someone copies the cert into github accidentally. No problem. We simple revoke and reissue.

Cert Expiration/Renew

We also have the ability to renew certs well ahead of their renewal time. This allows us to make decisions easily to renew a cert more often or less often depending on whatever is needed at the time. So if the service is less used, or SLA's/contracts have been updated this becomes an easy process. Certbot handles renewals automatically.

Can't anyone have a cert issued for a domain they don't own.

No, the LetsEncrypt will not issue a certificate if it can't verify the intended domain. To get around this a nefarious attacker would have to gain control of dns via spoofing/cache poisoning. Same threat vector as exist now.
